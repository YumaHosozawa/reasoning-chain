# 推論チェーン（マクロイベント→企業影響の多段推論）実装進捗レポート

**作成日**: 2026年4月  
**ステータス**: ロジック実装完了 / 検証フェーズ進行中

---

## 1. 概要

マクロ経済イベント（原油高騰・金利上昇・地政学リスク等）をテキストで入力すると、LLMが「一次→二次→三次→四次影響」を自動推論し、日本上場企業約4,000社の中から影響を受ける企業を自動特定するシステム。

人間のアナリストが見落としがちな二次・三次の間接影響まで網羅的にカバーすることが差別化ポイント。

---

## 2. システム全体像

```
マクロイベント（テキスト入力）
        │
        ▼
推論チェーン生成エンジン（LLM: Claude Opus）
        │ 一次〜四次影響をJSON構造で出力
        ▼
企業マッチングエンジン
        ├─ ベクトル検索（Qdrant）
        ├─ LLMスコアリング（Claude Haiku）
        └─ セグメント構成比照合
        │
        ▼
スコアリング結果（影響企業リスト）
        │
        ▼
Web UI / API / アラート出力
```

## 3. 実装ロジック詳細

### 3.1 推論チェーン生成

**処理の流れ**

1. マクロイベントのテキストをClaudeに入力
2. 以下の構造でJSONを生成させる

```
一次影響: イベントの直接的影響（例: 原油高→石油元売のマージン変動）
二次影響: 一次から派生（例: エネルギーコスト上昇→省エネ機器需要増）
三次影響: 二次から派生（例: 省エネ機器需要増→パワー半導体企業に追い風）
四次影響: 明確な因果がある場合のみ生成
```

3. 各影響ノードに以下の属性を付与
   - 影響の方向性（ポジティブ / ネガティブ / 混在）
   - 影響強度（高 / 中 / 低）
   - 因果関係の根拠テキスト
   - 関連キーワード（マッチング用）

**使用モデル**: Claude Opus 4.6（推論精度を優先）

---

### 3.2 企業プロファイルDB

**データソース**: EDINET（金融庁の開示システム）

- 日本上場企業約4,000社の有価証券報告書から以下を取得
  - 「事業の内容」セクション（非構造化テキスト）
  - セグメント情報（セグメント名・売上構成比・地域別比率）

**処理フロー**

```
EDINET API → 有報ZIP取得 → HTMLパース → テキスト抽出
    → LLM（Haiku）によるセグメント構造化
    → OpenAI text-embedding-3-large で埋め込みベクトル化（3072次元）
    → Qdrant（ベクトルDB）に格納
```

---

### 3.3 企業マッチング

影響ノード1件に対して以下の3ステップでスコアを算出し、上位企業を抽出する。

| ステップ | 処理内容 | 重み |
|---------|---------|------|
| ベクトル検索 | 影響説明をベクトル化し、Qdrantでコサイン類似度検索（上位50件） | 35% |
| LLMスコアリング | Claude Haikuが企業プロファイルと影響の関連度を0〜1で評価 | 40% |
| セグメント構成比 | 影響を受けるセグメントの売上構成比 | 25% |

**最終スコア式**
```
final_score = vector_similarity × 0.35
            + llm_relevance_score × 0.40
            + segment_exposure_ratio × 0.25
```

スコアが閾値（デフォルト: 0.6）以上の企業のみ出力。

## 4. 技術スタック

| レイヤー | 技術 | 用途 |
|---------|------|------|
| LLM（推論） | Claude Opus 4.6 | 推論チェーン生成 |
| LLM（スコアリング） | Claude Haiku 4.5 | 企業関連度スコアリング（高速・低コスト） |
| 埋め込みモデル | OpenAI text-embedding-3-large | 企業プロファイルと影響のベクトル化 |
| ベクトルDB | Qdrant | 企業プロファイルの類似度検索 |
| バックエンド | Python / FastAPI | REST API・推論ロジック |
| データ取得 | EDINET API v2 | 有価証券報告書の取得（APIキー不要） |
| 結果蓄積 | SQLite（SQLAlchemy） | 分析結果の永続化 |
| キャッシュ | Redis | LLMスコアリング結果のキャッシュ（重複呼び出し削減） |
| フロントエンド | Next.js 15 / TypeScript / Tailwind CSS | 試用UI |
| インフラ | Docker Compose | Qdrant・Redisのローカル環境構築 |

## 6. API仕様

| エンドポイント | メソッド | 概要 |
|--------------|---------|------|
| `/api/analyze` | POST | 推論チェーン生成 + 企業マッチング実行・結果DB保存 |
| `/api/analyze/stream` | GET | SSEで推論チェーンをストリーミング生成 |
| `/api/results` | GET | 過去の分析結果一覧取得 |
| `/api/results/{id}` | GET | 分析結果詳細取得 |

## 7. 出力例

```
イベント: 「トランプ政権が全輸入品に対して関税10%を発動」

影響チェーン:
  一次影響 [－ 輸出依存度の高い製造業]（影響度: 高）
    > 米国向け輸出品のコスト競争力が低下し、自動車・電機メーカーの収益悪化
  
  一次影響 [＋ 内需型・輸入代替企業]（影響度: 中）
    > 輸入品の価格上昇により国内生産品の競争力が相対的に向上
  
  二次影響 [＋ 国内サプライチェーン強化]（影響度: 中）
    > 国内調達回帰の動きが加速し、国内部品メーカーに恩恵
  
  三次影響 [＋ 物流・倉庫業]（影響度: 低）
    > 国内調達増加に伴う国内物流需要の増加

マッチング企業（上位）:
  ポジティブ影響:
    - XXXX（1234）: スコア 0.81 | 国内向け部品製造が主力
  ネガティブ影響:
    - YYYY（5678）: スコア 0.79 | 米国向け売上が全体の35%
```
